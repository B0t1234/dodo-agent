# 简历分析与候选人评估

## 关键规则

- **禁止输出任何过程性文字**。不要输出"我将要..."、"接下来我会..."、"已成功读取..."等描述你行为的文字。
- **读取完 JD 和简历后，立即直接输出完整报告**，不要做任何中间总结或计划说明。
- 报告必须使用中文，严格按照下方模板输出。
- 你的最终回复中应该**只包含报告本身**，不包含任何其他内容。

## 输入

1. **岗位 JD**：从项目根目录读取 `jd.md` 文件。如果不存在，从 `classpath:skills/resume-checker/jd.md` 读取。还是不存在，则提示用户创建，可参考 [jd-example.md](references/jd-example.md)。
2. **候选人简历**：用户提供的 PDF 文件路径，使用 read_file 工具读取。

## 五、建议面试问题

设计 8-12 个面试问题，分为以下类别。每道题必须包含：问题内容、考察目的、期望回答方向或追问方向。

### 5.1 技术能力验证（3-4题）
针对 JD 核心技术要求考察实际掌握程度。

### 5.2 项目经验深挖（2-3题）
针对简历中重点项目考察真实参与度和技术深度。

### 5.3 场景题（2-3题）
针对简历中重点项目给出特殊业务场景问题。

### 5.4 算法题（1-2题）
给出 1-2 道 LeetCode Hard 级别的算法题。

### 5.5 劣势/风险点考察（2-3题）
针对劣势或风险点设计验证问题，说明判断标准（加分/减分的回答）。

### 5.6 软技能与文化匹配（1-2题）
考察沟通能力、团队协作、学习能力等。
